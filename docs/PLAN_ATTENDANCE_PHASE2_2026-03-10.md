# 근태관리 Phase 2 구현 계획

- 기준일: 2026-03-10
- 대상 서비스: JHBioFarm 백오피스 `/attendance`
- 전제: 직원 수가 많지 않은 조직 기준으로, 복잡한 인사시스템이 아니라 "실제 운영 가능한 최소 근태관리"를 목표로 함

---

## 1. 목표

현재 Phase 1은 아래 수준까지 구현되어 있다.

- 개인 출근/퇴근 기록
- 월별 개인 근태 조회
- 관리자 전체 조회/검색/수정/삭제

Phase 2의 목표는 이를 아래 수준으로 끌어올리는 것이다.

- 휴가/반차를 포함해 결근과 부재를 구분할 수 있는 상태 판정
- 관리자가 오늘 상황을 한눈에 볼 수 있는 현황판
- 직원이 누락된 출퇴근 기록을 직접 정정 요청할 수 있는 흐름
- 회사 기준 근무시간을 바탕으로 자동 상태 판정

즉, Phase 2는 "출퇴근 기록 저장"에서 "운영 가능한 근태관리"로 가는 단계다.

---

## 2. 핵심 원칙

### 2.1 최소 범위 우선

직원 수가 적은 조직 기준으로 아래만 먼저 구현한다.

- 근무 기준 설정 1세트
- 휴가/반차 신청 및 승인
- 자동 상태 판정
- 관리자 오늘 현황판
- 정정 요청

아래 항목은 Phase 2 범위에서 제외한다.

- 부서별 근무제
- 유연근무/시차출근
- GPS/IP 기반 출근 인증
- 연장근무 승인 플로우
- 연차 자동 발생/차감
- 급여 시스템 연동

### 2.2 휴가가 근태보다 먼저 반영되어야 함

근태 상태 판정은 항상 아래 우선순위를 따른다.

1. 승인된 휴가/반차/병가/공가
2. 실제 출근/퇴근 기록
3. 결근/지각/조퇴 자동 판정

이 원칙이 없으면 휴가를 결근으로 잘못 잡게 된다.

### 2.3 관리자는 "오늘 상태"를 먼저 봐야 함

월별 표보다 먼저 필요한 것은 오늘 누가

- 출근했는지
- 늦었는지
- 휴가인지
- 퇴근했는지

를 한눈에 보는 것이다.

---

## 3. Phase 2 범위

## 3.1 근무 기준 설정

### 목적

자동 상태 판정의 기준값을 저장한다.

### 화면

- 신규: `/attendance/settings`

### 설정 항목

- 기본 출근 시간
- 기본 퇴근 시간
- 지각 허용 분
- 점심시간 차감 분
- 기준 근무시간

### 예시

- 출근: `09:00`
- 퇴근: `18:00`
- 지각 허용: `10분`
- 점심 차감: `60분`
- 기준 근무시간: `8시간`

### 권한

- `admin`만 수정 가능
- `modifier`, `viewer`는 조회만 가능하거나 접근 제한

---

## 3.2 휴가/반차 신청 및 승인

### 목적

부재를 결근과 구분한다.

### 화면

- `/attendance/records`
  - 휴가 신청 버튼
  - 내 신청 내역
- `/attendance/admin`
  - 휴가 신청 승인/반려 영역

### 초기 휴가 타입

- 연차
- 오전 반차
- 오후 반차
- 병가
- 공가
- 기타

### 동작 흐름

1. 직원이 휴가를 신청
2. 관리자가 승인/반려
3. 승인된 건은 근태 상태 판정에 반영

### Phase 2에서 하지 않는 것

- 연차 잔여일 자동 차감
- 근속연수별 연차 계산
- 복수 승인자 결재

---

## 3.3 자동 상태 판정

### 목적

출근/퇴근 시간을 운영 상태로 변환한다.

### 상태값

- 정상
- 지각
- 조퇴
- 결근
- 휴가
- 오전 반차
- 오후 반차
- 병가
- 공가

### 판정 규칙

#### 기본 규칙

- 승인된 휴가가 있으면 휴가 상태 우선
- 출근 기록이 없고 승인 휴가도 없으면 결근
- 출근 시각이 기준 출근 시각 + 허용분보다 늦으면 지각
- 퇴근 시각이 기준 퇴근 시각보다 빠르면 조퇴

#### 반차 규칙

- 오전 반차: 오전 결근/지각 판정 제외, 오후 출근만 확인
- 오후 반차: 오후 조퇴 판정 제외, 오전 출근 여부만 확인

### 표시 위치

- `/attendance/records`
- `/attendance/admin`
- 관리자 오늘 현황판

---

## 3.4 관리자 오늘 현황판

### 목적

오늘 근태 상태를 한 번에 확인한다.

### 화면

- `/attendance/admin` 상단

### 표시 카드

- 오늘 출근 인원
- 근무중
- 퇴근 완료
- 지각
- 결근
- 휴가/반차

### 하단 테이블 컬럼 예시

- 이름
- 아이디
- 오늘 상태
- 출근 시간
- 퇴근 시간
- 근무시간
- 부재 유형
- 관리 액션

### 핵심 효과

현재 월별 표 중심 구조를 "오늘 운영 중심"으로 전환한다.

---

## 3.5 정정 요청

### 목적

누락/오기록을 사용자 요청 기반으로 정리한다.

### 요청 타입

- 출근 누락 정정
- 퇴근 누락 정정
- 시간 수정 요청

### 화면

- `/attendance/records`
  - 정정 요청 버튼
  - 내 요청 내역
- `/attendance/admin`
  - 정정 요청 승인/반려 영역

### 동작 흐름

1. 직원이 날짜/시간/사유를 입력
2. 관리자가 승인/반려
3. 승인 시 `attendance_records` 반영
4. 요청 이력은 별도 보존

## 6. API/조회 규칙

## 6.1 개인 화면

- 본인 `attendance_records`만 조회
- 본인 `leave_requests`만 조회
- 본인 `attendance_adjustment_requests`만 조회

## 6.2 관리자 화면

- 전체 `attendance_records` 조회
- 전체 `leave_requests` 승인/반려
- 전체 `attendance_adjustment_requests` 승인/반려

## 6.3 상태 계산 우선순위

1. 승인된 휴가/반차
2. 출퇴근 기록
3. 설정된 근무 기준

---

## 7. 구현 순서

### Step 1

`attendance_settings`

- 테이블 생성
- `/attendance/settings` 화면 생성
- 관리자만 수정 가능

### Step 2

`leave_requests`

- 휴가 신청/승인 플로우
- `/attendance/records`, `/attendance/admin` 반영

### Step 3

자동 상태 판정

- `attendance_records + attendance_settings + leave_requests` 조합으로 상태 계산
- 개인/관리자 화면에 상태 표시

### Step 4

관리자 오늘 현황판

- `/attendance/admin` 상단 카드
- 오늘 상태 테이블

### Step 5

`attendance_adjustment_requests`

- 정정 요청/승인 플로우
- 승인 시 근태 기록 반영

---

## 8. 완료 기준

아래가 되면 Phase 2 완료로 본다.

- 근무 기준 1세트 저장 가능
- 연차/오전 반차/오후 반차/병가/공가/기타 신청 가능
- 관리자 승인/반려 가능
- 휴가 승인건이 결근으로 잡히지 않음
- 지각/조퇴/결근이 자동 표시됨
- 관리자가 오늘 현황을 카드+테이블로 볼 수 있음
- 직원이 정정 요청을 올리고 관리자가 승인/반려 가능

---

## 9. 비범위

이번 단계에서 하지 않는다.

- 팀/부서별 근무제
- GPS/IP 출근 인증

- 급여 연동

# Goodforpet Change Log

굿포펫 관련 수정 사항을 계속 누적 기록하는 작업 로그입니다.

## 2026-03-19

### 실구매 추이 성장률 표시 추가

- 대시보드 `실구매 추이` 차트는 `실구매 건수`만 보이도록 유지
- 성장률은 별도 선 그래프 대신 `1주차 -> 2주차`, `2일 -> 3일` 같은 구간 사이에 바로 붙는 흐름형 표시로 변경
- 성장률 계산은 이전 구간 대비 `(현재 - 이전) / 이전 * 100` 기준
- 전체 기간 기준 마지막 달은 현재 달이 아니라 `실데이터가 있는 가장 최근 달` 기준으로 맞춤

### 상품 구매 추이 페이지 추가

- 매출분석 사이드바에 `상품 구매 추이` 메뉴 추가
- 새 페이지:
  - 상품 목록 전체 나열
  - 상품명과 옵션을 함께 표시하고 옵션명으로도 검색 가능하게 수정
  - 상품 클릭 시 기간별 실구매 수량 추이 표시
  - 차트는 실구매 수량만 표시
  - 성장률은 구간별 라벨 사이에 바로 붙는 흐름형 표시로만 노출
  - 상단/섹션 설명 문구는 걷어내고 제목과 데이터 위주로 단순화
- 선택한 분석 기간/주차 기준으로 같은 기간 필터를 그대로 사용

### 고객 성장 단계 기준 변경

- 고객 성장 단계를 `누적 구매월 수` 기준으로 변경
- 단계 기준:
  - 신규: 1~2개월
  - 성장: 3~5개월
  - 단골: 6~11개월
  - 핵심: 12개월 이상
- 선택한 월은 화면 노출 범위만 결정하고, 단계 계산은 전체 실구매 이력 기준으로 계산하도록 수정

### 구매 강도 추가

- 최근 90일 `고유 구매일 수` 기준으로 구매 강도 도입
- 강도 기준:
  - 휴면: 0일
  - 낮음: 1일
  - 보통: 2일
  - 높음: 3~4일
  - 매우 높음: 5일 이상
- 고객분석 화면에 구매 강도 컬럼, 상세 표시, 검색/필터, 엑셀 다운로드 반영

### 고객 성장 단계 화면 정리

- 화면 구조를 3섹션 중심으로 단순화
- 상단 히어로 설명과 승급 후보 보조 설명 문구 제거
- 유지 섹션:
  - 단계 카드 4개
  - 성장 단계 분포 + 전환 현황
  - 승급 후보
- 제거 섹션:
  - 행동 비교 차트
  - 최근 승급 고객
  - 단계별 인기 상품

### 이탈 위험 로직 개편

- 기존 `최근 주문일 90일 초과` 고정 기준 제거
- 새 기준:
  - `오늘 - 마지막 구매일 > 예상 소비일`
  - 마지막 구매 이후 재구매 없음
  - 마지막 구매일 상품들 중 `가장 긴 예상 소비일` 기준
- 예상 소비일 미입력 상품만 마지막 구매에 포함된 고객은 `판단 제외` 처리
- 고객분석 화면에서 이탈 상태를 `위험 / 정상 / 판단 제외`로 표시하도록 수정
- 대시보드와 성장 단계 요약도 같은 기준을 쓰도록 통일

### 이탈 위험 안전장치 추가

- 상품 마스터에 `expected_consumption_days`가 하나도 설정되지 않은 경우
  - 고객분석
  - 대시보드
  - 성장 단계 요약
  에서 이탈 위험을 강제로 만들지 않도록 안전장치 추가
- 즉, 전체 상품 기준 예상 소비일 설정이 0건이면 고객은 모두 `판단 제외`로 처리
- 추가 보강:
  - 로컬 개발 서버 HMR 상태에서 예전 이탈 위험 값이 메모리에 남아 있어도
    고객분석/대시보드가 즉시 `판단 제외`/`0명`으로 정리되도록 화면 상태 정리 로직 추가
- 표시 레이어 보강:
  - 고객분석 배지/필터/상세 표시는 `expected_consumption_days` 설정이 0건이면 계산 상태와 무관하게 무조건 `판단 제외`로 표시
  - 대시보드 KPI/리스트도 설정 0건이면 무조건 `0명/빈 목록`으로 표시
- 실제 원인 수정:
  - 프론트 메타 로딩에서 `expected_consumption_days = null` 값이 `1일`로 잘못 정규화되던 버그 확인
  - 원인: `Number(null) === 0`인데 이를 유효 숫자로 처리하고 `Math.max(1, 0)`을 적용하고 있었음
  - 조치: 공통 `normalizeExpectedConsumptionDays()` 함수로 정규화 로직 통일
  - 결과: `null`, `undefined`, `0`은 모두 `판단 제외`로만 흐르도록 수정

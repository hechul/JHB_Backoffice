## 1) 프로젝트 요약

### 1.1 제품 목적
스마트스토어 **주문 데이터**와 **체험단 데이터**를 업로드하여,
- 고객을 자동 분류(체험단/실구매/확인 필요/이탈 위험)
- 운영자가 결과를 검토·수정(수동 확정)하고
- 고객 분석/엑셀 추출로 마케팅 업무까지 이어지게 하는 **백오피스**.

### 1.2 주요 사용자(페르소나)
- 운영 관리자(admin): 월별 업로드 → 분석 실행 → 수동 확인 → 고객 추출
- 대표/의사결정자(admin): 대시보드 KPI/이탈 위험 현황 확인
- 조회 사용자(viewer): 결과/이력 조회 중심(실행/수정 불가)

## 3) MVP 기능 우선순위

### Must (MVP 필수)
- 인증/권한: 로그인/로그아웃, admin/viewer 분기
- 월 선택기(기간 선택): 월 기준 데이터 조회(Upload/Filter는 all에서 실행 금지)
- 업로드: 주문/체험단 업로드, 필수 컬럼 검증, 업로드 결과 요약, 상품 매핑
- 필터링: 실행/진행률, 결과 탭, 수동 확정(저장/적용), 정리 파일 다운로드
- 고객 분석: 검색/필터, 상세 보기, 엑셀 다운로드
- 실행 이력: 필터링 실행 이력 + 수동 변경 이력
- 상품 목록: 상품 등록/수정/삭제(필수 속성: 펫타입/성장단계/라인/옵션)
- 계정 관리: 사용자 초대, 권한 변경, 비활성화(삭제는 정책 확정 필요)

### Should (1차 확장)
- 업로드 템플릿(양식) 다운로드 고도화/가이드
- URL 쿼리 동기화(대시보드 → 고객분석 딥링크, 고객분석 필터↔URL)
- 알림 클릭 시 관련 페이지로 이동

### Could (후속)
- 자동 리포트/고급 대시보드 확장
- 일괄 분류/동시성 Lock 등은 운영 규모(1~2명)에서는 후순위

## 5) 전역 규칙(공통 UX/정책)

### 5.1 공통 UX 원칙
- 기존 디자인 시스템 유지(레이아웃/인터랙션만 정돈)
- 각 페이지 상단에 “마지막 실행/처리 상태/기준 월”을 우선 배치(현재 상태를 빠르게 판단)
- 1페이지 1개 우선 CTA:
  - Upload: 업로드 시작
  - Filter: 필터링 시작
  - Customers: 엑셀 다운로드
  - Products: 신규 상품 등록
  - Users: 초대하기

### 5.2 권한 규칙
- admin: 전체 기능
- modifier: 데이터 작업 권한(admin과 동일) + 계정 초대/권한 변경 불가
- viewer: 조회 전용(실행/수정/삭제 버튼 숨김 또는 비활성), 일부 페이지 진입 차단 가능(Upload/Filter/Products/Users 등)

### 5.3 기간(월 선택기) 규칙
- 노출: `/dashboard`, `/customers`, `/logs`, `/upload`, `/filter`
- `all` 선택 시:
  - `/upload`, `/filter` 실행 금지(조회만 가능)
  - `/dashboard`, `/customers`, `/logs`는 조회 허용

### 5.4 공통 상태 패턴(필수)
- Loading / Empty / Error / Default(데이터 존재)
- 업로드/필터링 같은 장시간 작업은 동일 패턴:
  - 실행 버튼 → (필요 시) 확인 모달 → 진행 표시(배너+Progress) → 완료 토스트/알림 → 결과 상태 전환

## 6) 사용자 플로우(정상 + 예외)

### FL-01 로그인 → 홈 → 모듈 진입
- 정상: `/login` → `/` → (매출 분석→`/dashboard`) 또는 (설정→`/settings/users`)
- 예외: 인증 실패 안내 / 권한 부족 안내 / 네트워크 오류 재시도

### FL-02 월간 운영 메인 플로우
- 정상: 업로드(`/upload`) → 검증/매핑 → 필터링(`/filter`) 실행 → 확인 필요 수동 확정 → 고객 분석(`/customers`) → 엑셀 추출
- 예외:
  - 업로드 형식/필수 항목 누락 → 실패 사유 표시 → 재업로드
  - 매핑 미완료 → 분석 실행 불가 + 남은 건수 표시
  - 분석 실패 → 실패 안내 + `/logs`에서 원인 확인 + 재실행

### FL-03 대시보드 → 고객 분석 딥링크
- 정상: KPI/버튼 클릭 → `/customers?...`로 이동(조건 자동 적용)
- 예외: 결과 0건 → Empty 안내 + 조건 초기화/업로드 유도

### FL-04 고객 분석 → 엑셀 다운로드
- 정상: 필터 적용 → 엑셀 다운로드(현재 조건 기준)
- 예외: 생성 실패 → 재시도 안내

### FL-05 실행 이력
- 정상: `/logs`에서 실행/수동 변경 이력 확인(상세 펼침)
- 예외: 이력 없음(Empty) / 조회 실패(Error)
